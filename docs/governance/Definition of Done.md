# Definition of Done

Date: 2026-07-10

A change is done when all of the following are true:

- scope matches the approved task
- no prohibited product behavior was introduced
- required docs are updated
- local quality gates pass
- CI is green
- Security workflow is green when applicable
- review comments are addressed
- ADR or RFC artifacts exist when required
- release-impacting changes include changelog consideration
- security and observability impacts are documented
